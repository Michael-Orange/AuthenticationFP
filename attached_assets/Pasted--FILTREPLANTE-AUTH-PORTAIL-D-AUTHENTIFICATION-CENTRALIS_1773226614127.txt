 🔐 FILTREPLANTE-AUTH - PORTAIL D'AUTHENTIFICATION CENTRALISÉE

## OBJECTIF
Créer un portail d'authentification unique (SSO) pour toutes les applications internes FiltrePlante. Cette app gère la connexion centralisée, la vérification des permissions et la génération de tokens SSO pour rediriger les utilisateurs vers les apps autorisées.

## CONTEXTE BUSINESS
Actuellement, FiltrePlante a plusieurs apps internes (Stock, Prix Référentiel, Factures, Maintenance). Les utilisateurs ont des accès différenciés selon leur rôle. Cette app Auth centralisée permet :
- ✅ Un seul login pour toutes les apps (SSO)
- ✅ Gestion centralisée des permissions
- ✅ Expérience utilisateur fluide (pas de re-login entre apps)
- ✅ Architecture scalable pour futures apps

## STACK TECHNIQUE

**Frontend :**
- React + TypeScript + Vite
- Tailwind CSS + FiltrePlante Design System

**Backend :**
- Node.js + Express
- PostgreSQL (Neon) - Base de données partagée
- JWT (jsonwebtoken) pour authentification
- CryptoJS pour hashage mot de passe (compatibilité avec apps existantes)
- cookie-parser pour gestion sessions

**Hébergement :**
- Replit Autoscale
- URL cible : `auth-filtreplante.replit.app`

## ARCHITECTURE GLOBALE

```
FiltrePlante-Auth (auth-filtreplante.replit.app)
├── /login                    → Page de connexion (username/password)
├── /dashboard                → Mes applications (après login)
├── /api/auth/login           → Endpoint connexion
├── /api/auth/logout          → Endpoint déconnexion
├── /api/auth/me              → Vérifier session Auth courante
├── /api/sso/generate         → Générer token SSO pour une app
└── Session Auth (cookie)     → auth_session (7 jours)

Apps clientes (Stock, Prix, Factures, etc.)
├── Détecte pas de session → Redirect vers Auth
├── /sso/login?token=XXX   → Endpoint réception token SSO
└── Valide token → Crée session app
```

## FLOW UTILISATEUR COMPLET

### Scénario 1 : Premier accès à une app (ex: Stock)

1. User tape `stock.replit.app`
2. Stock détecte : Pas de cookie `auth_token`
3. Stock redirige vers `auth-filtreplante.replit.app/login?redirect=stock`
4. Auth affiche page login
5. User entre username/password
6. Auth vérifie credentials + permissions pour "stock"
7. Si autorisé :
   - Crée session Auth (cookie `auth_session` sur auth-filtreplante.replit.app)
   - Génère token JWT SSO
   - Redirige vers `stock.replit.app/sso/login?token=XXX`
8. Stock valide token et crée session locale
9. ✅ User connecté sur Stock

### Scénario 2 : Accès à une autre app (ex: Prix) - SSO automatique

1. User tape `prix.replit.app`
2. Prix détecte : Pas de cookie `auth_token`
3. Prix redirige vers `auth-filtreplante.replit.app/login?redirect=prix`
4. **Auth détecte : Session Auth valide (cookie auth_session existe)**
5. **Pas de page login affichée !**
6. Auth génère directement token JWT SSO pour "prix"
7. Redirige vers `prix.replit.app/sso/login?token=XXX`
8. Prix valide token et crée session locale
9. ✅ User connecté sur Prix (SANS retaper mot de passe)

### Scénario 3 : Dashboard "Mes Applications"

1. User connecté sur Auth clique "Mes Applications"
2. Auth affiche cartes des apps autorisées selon `user.apps`
3. User clique sur une app (ex: "Factures")
4. Auth génère token SSO
5. Redirige vers `factures.replit.app/sso/login?token=XXX`
6. ✅ Accès direct sans login

### 4. ENDPOINTS BACKEND

#### POST /api/auth/login

**Entrée :**
```json
{
  "username": "michael",
  "password": "password123"
}
```

**Logique :**
1. Vérifier username existe dans `referentiel.users`
2. Comparer password avec `password_hash` (CryptoJS)
3. Si credentials valides :
   - Créer cookie `auth_session` (httpOnly, 7 jours)
   - Payload : `{ userId, username, nom, role, apps }`
   - Retourner user info
4. Si échec : 401 "Identifiants invalides"

**Sortie (succès) :**
```json
{
  "success": true,
  "user": {
    "id": 1,
    "username": "michael",
    "nom": "Michael",
    "role": "admin",
    "apps": ["stock", "prix"]
  }
}
```

**Implémentation :**
```typescript
router.post('/api/auth/login', async (req, res) => {
  const { username, password } = req.body;
  
  try {
    // 1. Récupérer user
    const result = await db.query(
      'SELECT * FROM referentiel.users WHERE username = $1',
      [username]
    );
    
    if (result.rows.length === 0) {
      return res.status(401).json({ error: "Identifiants invalides" });
    }
    
    const user = result.rows[0];
    
    // 2. Vérifier password (CryptoJS - compatibilité apps existantes)
    const isPasswordValid = verifyPassword(password, user.password_hash);
    if (!isPasswordValid) {
      return res.status(401).json({ error: "Identifiants invalides" });
    }
    
    // 3. Mettre à jour last_login
    await db.query(
      'UPDATE referentiel.users SET last_login = NOW() WHERE id = $1',
      [user.id]
    );
    
    // 4. Créer session Auth (cookie de l'app Auth)
    const sessionPayload = {
      userId: user.id,
      username: user.username,
      nom: user.nom,
      role: user.role,
      apps: user.apps
    };
    
    const sessionToken = jwt.sign(sessionPayload, process.env.JWT_SECRET!, {
      expiresIn: '7d'
    });
    
    res.cookie('auth_session', sessionToken, {
      httpOnly: true,
      secure: process.env.NODE_ENV === 'production',
      sameSite: 'lax',
      maxAge: 7 * 24 * 60 * 60 * 1000 // 7 jours
    });
    
    // 5. Retourner user info (sans password_hash)
    res.json({
      success: true,
      user: {
        id: user.id,
        username: user.username,
        nom: user.nom,
        role: user.role,
        apps: user.apps
      }
    });
    
  } catch (error) {
    console.error('Login error:', error);
    res.status(500).json({ error: "Erreur serveur" });
  }
});
```

#### GET /api/sso/generate?app=stock

**But :** Générer token SSO pour rediriger vers une app cible

**Paramètres :**
- `app` (query) : ID de l'app cible ("stock", "prix", "factures", etc.)

**Logique :**
1. Vérifier session Auth valide (cookie auth_session)
2. Vérifier permissions : user.apps contient `app` OU role === 'admin'
3. Si autorisé : Générer token JWT SSO (même structure, expiration courte 5 min)
4. Si refusé : 403 "Accès refusé"

**Sortie :**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "redirectUrl": "https://stock.replit.app/sso/login"
}
```

**Implémentation :**
```typescript
router.get('/api/sso/generate', async (req, res) => {
  const { app } = req.query;
  
  if (!app) {
    return res.status(400).json({ error: "Paramètre 'app' requis" });
  }
  
  // 1. Vérifier session Auth
  const sessionToken = req.cookies.auth_session;
  if (!sessionToken) {
    return res.status(401).json({ error: "Non authentifié" });
  }
  
  try {
    // 2. Décoder session
    const user = jwt.verify(sessionToken, process.env.JWT_SECRET!) as any;
    
    // 3. Vérifier permission pour l'app demandée
    const hasAccess = user.apps.includes(app) || user.role === 'admin';
    if (!hasAccess) {
      return res.status(403).json({ 
        error: `Vous n'avez pas accès à ${app}. Contactez l'administrateur.` 
      });
    }
    
    // 4. Trouver URL de l'app
    const availableApps = JSON.parse(process.env.AVAILABLE_APPS || '[]');
    const targetApp = availableApps.find((a: any) => a.id === app);
    
    if (!targetApp) {
      return res.status(404).json({ error: "Application inconnue" });
    }
    
    // 5. Générer token SSO (expiration courte : 5 min)
    const ssoToken = jwt.sign(
      {
        userId: user.userId,
        username: user.username,
        nom: user.nom,
        role: user.role,
        apps: user.apps
      },
      process.env.JWT_SECRET!,
      { expiresIn: '5m' } // Token SSO expire rapidement
    );
    
    res.json({
      token: ssoToken,
      redirectUrl: `${targetApp.url}/sso/login`
    });
    
  } catch (error) {
    console.error('SSO generation error:', error);
    res.status(401).json({ error: "Session invalide" });
  }
});
```

#### GET /login?redirect=stock

**But :** Page de login avec redirection automatique après succès

**Logique frontend :**
```typescript
// Extraire paramètre redirect
const params = new URLSearchParams(window.location.search);
const redirectApp = params.get('redirect'); // "stock"

// Après login réussi
const handleLoginSuccess = async () => {
  if (redirectApp) {
    // Générer token SSO et rediriger
    const response = await fetch(`/api/sso/generate?app=${redirectApp}`);
    const { token, redirectUrl } = await response.json();
    
    if (response.ok) {
      window.location.href = `${redirectUrl}?token=${token}`;
    } else {
      // Erreur permission
      setError("Vous n'avez pas accès à cette application");
    }
  } else {
    // Pas de redirect, aller au dashboard
    navigate('/dashboard');
  }
};
```

**Cas particulier - Session Auth déjà valide :**
```typescript
// Au chargement de /login
useEffect(() => {
  const checkExistingSession = async () => {

        // Générer token SSO et rediriger directement
        const ssoResponse = await fetch(`/api/sso/generate?app=${redirectApp}`);
        const { token, redirectUrl } = await ssoResponse.json();
        window.location.href = `${redirectUrl}?token=${token}`;
      } else {
        navigate('/dashboard');
      }
    }
  };
  
  checkExistingSession();
}, []);
```

### 5. SÉCURITÉ

**Cookies :**
```typescript
// Cookie session Auth (pour l'app Auth)
res.cookie('auth_session', token, {
  httpOnly: true,           // Protection XSS
  secure: true,             // HTTPS only en production
  sameSite: 'lax',         // Protection CSRF
  maxAge: 7 * 24 * 60 * 60 * 1000  // 7 jours
});
```

**JWT :**
- Session Auth : Expiration 7 jours
- Token SSO : Expiration 5 minutes (usage unique)
- Secret partagé avec toutes les apps (JWT_SECRET)

**Password hashing (CryptoJS - compatibilité) :**
```typescript
import CryptoJS from 'crypto-js';

function verifyPassword(plainPassword: string, hash: string): boolean {
  const computedHash = CryptoJS.SHA256(plainPassword).toString();
  return computedHash === hash;
}
```

**Protection CORS :**
```typescript

const allowedOrigins = [
```
